TheHive Certificate Verification
================================

When the ``useSSL`` property is set to ``true`` in the server configuration file
(see :ref:`Service Configuration File <dxl_service_config_file_label>`), TheHive
DXL service communicates to TheHive server over SSL/TLS. TheHiveDXL service
supports validating the authenticity of the certificate associated with TheHive
server that is being connected to. Enabling and disabling of this capability is
controlled via the ``verifyCertificate`` property in the server configuration
file. While this validation can be disabled, it is highly recommended that it be
enabled for production environments.

The remainder of this page walks through the steps necessary to obtain the
information required to perform this validation successfully.

Two types of certificate validation take place when connecting to TheHive server:

1. The ``host`` as defined in the service configuration file for TheHive server
   must match the hostname in TheHive server's certificate.
2. TheHive server's certificate must be signed by a Certificate Authority (CA)
   that can be found in the CA bundle (specified via the ``verifyCertBundle``
   property) associated with TheHive server in the service configuration file.

To determine the hostname in TheHive server's certificate and the certificate of
the signing CA you can execute the following command via ``openssl``.

    .. parsed-literal::

        openssl s_client -showcerts -connect <thehive-host>:<thehive-port>

For example:

    .. parsed-literal::

        openssl s_client -showcerts -connect thehivetestsystem:9443

In the output above, the two important portions for validating TheHive server's
certificate have been highlighted in bold.

1. The first portion includes the common name (CN) associated with TheHive
   server's certificate.

   In this particular example the CN is as follows: ``CN=thehivetestsystem``

   Based on this, for hostname validation to succeed the ``host`` value for
   TheHive server as defined in the service configuration file must have a value
   of ``thehivetestsystem``.

2. The second portion highlighted is the certificate of the CA that was used to
   sign TheHive server's certificate.

   For certificate validation to succeed, this certificate text must be copied
   (including the `BEGIN` and `END` lines) into a new file. Next, the
   ``verifyCertBundle`` value for TheHive server as defined in the service
   configuration file must have a path to this newly created file.
